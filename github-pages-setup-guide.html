GitHub Pages

把仓库首页发布成在线网页

这份指南用于手动开启 GitHub Pages。核心机制很简单：GitHub 会把你指定的分支和目录当作网站根目录，目录里的 index.html 会成为默认首页。

仓库 gavinssr/myknowledge
发布源 main / root
在线地址 gavinssr.github.io/myknowledge/

手动设置步骤

  1. 打开仓库页面

    进入 github.com/gavinssr/myknowledge。

  2. 进入 Settings

    点击仓库顶部导航里的 Settings。

  3. 打开 Pages 设置

    在左侧栏找到并点击 Pages。

  4. 选择发布来源

    在 Build and deployment 里，把 Source 设置为 Deploy from a branch。

  5. 选择分支和目录

    Branch 选择 main，目录选择 / (root)，然后点击 Save。

  6. 等待发布完成

    首次发布通常需要几十秒到几分钟。完成后访问 https://gavinssr.github.io/myknowledge/。

以后更新页面时，只需要把新的文件提交并推送到 main。GitHub Pages 会自动重新发布，但线上页面可能有短暂缓存延迟。

发布前检查

  • 仓库根目录存在 index.html。
  • 首页里链接的 HTML、图片、PDF 等资源已经提交到仓库。
  • 文件路径使用相对路径，例如 Le-Renting/outputs/.../report.html。
  • 仓库是 public，或当前 GitHub 账号套餐允许 private repository 使用 Pages。
