1. CEO Review

CEO/Founder-Mode Plan Review

切换到创始人视角审查方案——我们是否在解决正确的问题？ambition 是否足够？

四种模式

SCOPE EXPANSION — 大胆设想 HOLD SCOPE — 最大严谨 SCOPE REDUCTION — 精简核心 SELECTIVE — 精选扩展

触发方式

在 Hermes 对话中直接说：

用 gstack CEO review 审查这个方案
我会先用 clarify 确认审查模式，然后读取你的方案文档或听你描述，输出结构化审查报告。

2. Engineering Review

Architecture & Technical Review

审查技术方案的正确性和完整性——架构、数据流、边界case、安全、性能、可测试性、可维护性。

审查七维度

  • 架构设计 — 模块划分、耦合度、单点故障
  • 数据流 — 状态管理、一致性、异步处理
  • 边界 Case — 空/错误/加载状态、并发、极端输入
  • 安全性 — SQL注入、XSS、认证、密钥泄露、路径遍历
  • 性能 — 瓶颈识别、缓存策略、扩展方案
  • 可测试性 — 关键路径覆盖、测试数据管理
  • 可维护性 — 代码组织、错误信息、日志监控
用 gstack eng review 审查这个架构

3. AutoPlan

One-Click Full Review Pipeline

一键运行 CEO → 设计 → 工程 → DX 四层审查。用六条决策原则自动处理常规判断，仅在关键分歧点询问。

决策六原则

  1. 安全优先 — 有风险→升级
  2. 可逆性 — 易回滚→自动
  3. 影响面 — <3文件→自动
  4. 先例 — 有先例→自动
  5. 成本 — <1小时→自动
  6. 确定性 — 共识→自动
用 gstack autoplan 审查这个方案

4. Design Consultation

Complete Design System Builder

从零构建完整设计系统，输出 DESIGN.md。涵盖美学方向、颜色、字体、间距、布局、圆角阴影、动效。

九步流程

  • 产品理解 → 美学方向 → 颜色系统 → 字体系统
  • 间距系统 → 布局系统 → 圆角阴影 → 动效系统 → 组件预览

可配合 Hermes 的 baoyu-infographic 和 design-md 技能生成可视化预览。

用 gstack design-consultation 构建设计系统

5. Review (PR Review)

Pre-Landing Code Review

上线前深度代码审查，发现 CI 通过但生产会炸的问题。覆盖安全、数据完整性、边界、LLM 特定、副作用。

严重级别

CRITICAL — 必须修复 WARNING — 应该修复 SUGGESTION — 建议改进
用 gstack review 审查当前 PR

可配合加载 github-code-review skill 使用

6. QA

Browser Quality Assurance

用 Hermes 原生 browser_navigate/click/snapshot/vision/console 工具在真实浏览器中测试。

测试流程

  • 环境准备 → Happy Path → Edge Cases
  • 响应式检查 → Console 检查 → Bug 报告 → 修复验证
Hermes 的 browser_* 工具是原生支持的，比 gstack 的 headless Chromium 更强大——支持 snapshot、vision 截图分析、console 日志捕获。
用 gstack QA 测试 http://localhost:3000

7. Investigate

Systematic Root-Cause Debugging

四阶段调试：复现 → 隔离 → 假设 → 验证。核心原则：先理解，再修复。

常见陷阱

  • 跳过复现 → 打地鼠
  • 同时追多个假设 → 方向混乱
  • 改代码"测试" → 掩盖根因
  • 忽略"不相关"信息 → 80%根因在那
用 gstack investigate 排查登录超时问题

可配合加载 systematic-debugging skill

8. Ship

Test → Review → Push → PR Pipeline

六步发布：本地验证 → 运行测试 → 代码审查 → 推送 → 创建 PR → 监控 CI。

安全检查

  • 无 console.log / print 调试代码
  • 无硬编码密钥/token
  • 敏感文件在 .gitignore
  • commit message 清晰
用 gstack ship 发布

9. Design HTML

Production-Quality HTML/CSS Prototypes

生成自包含、响应式、暗色模式、可访问的 HTML 原型文件，可直接浏览器打开。

设计原则

  • 自包含 — 所有 CSS/JS 内联
  • 响应式 — mobile/tablet/desktop
  • 暗色模式 — prefers-color-scheme
  • 可访问 — 语义化 HTML + ARIA
  • 生产质量 — 可直接部署
用 gstack design-html 生成一个 SaaS landing page

可配合 baoyu-infographic、architecture-diagram 技能

10. Retro

Weekly Retrospective

系统化回顾工作，涵盖交付节奏、代码质量、决策质量、工具流程、学习成长五个维度。

输出结构

  • 完成事项
  • What Went Well
  • What Could Be Better
  • Action Items（带负责人+截止日期）
  • 数据洞察（PR数、bug数、交付周期）
用 gstack retro 做周回顾
